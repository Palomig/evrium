# Спецификация: интерактивный сайт по геометрии (Атанасян, 7–9 кл.)

Ниже — полная спецификация, технический промт для нейросети, JSON-шаблоны упражнений и список интерактивных сценариев по главам учебника. Файл готов к использованию — скопируйте `technical prompt` и вставьте в выбранную вами нейросеть для генерации кода.

---

## Краткое резюме (требования и ограничения)

- **Тип хостинга:** shared hosting без возможности запускать bash / устанавливать сервисы. Поэтому **нужен полностью статический сайт** (HTML + CSS + JS).
- **Интерактивность:** реализуется на клиенте через **SVG + vanilla ES6 JavaScript (модули)**.
- **Зависимости:** только CDN-подключения (если нужны). Никаких `npm`, сборки, серверного кода.
- **Редактируемость:** упражнения описываются JSON-файлами (`/exercises/*.json`). Встроенный `Editor mode` для правки JSON и live-preview.
- **Цель:** дать технический промт для нейросети и набор сценариев, покрывающих темы учебника Атанасяна для 7–9 классов.

---

## Короткое техническое решение (обоснование)

1. **Статический стек:** HTML/CSS/JS, ES6 модули. Рендеринг геометрии — через SVG.
2. **Файловая организация:** простая, чтобы можно было просто загрузить файлы на шаред-хостинг.
3. **Редактирование:** JSON-формат для упражнений + визуальный редактор (WYSIWYG + raw JSON) для быстрой правки учителем.
4. **Интерактивные виджеты:** draggable точки, ползунки, измерители углов и площадей — все на клиенте.
5. **Мобильность и доступность:** поддержка touch и клавиатуры.

## Технический промт (скопируйте весь блок и вставьте в нейросеть)

> **Задача:** Сгенерируй полностью статический веб-сайт для интерактивного обучения геометрии (по учебнику Л. С. Атанасяна, 7–9 кл.).
>
> **Ограничения:**
> 1. Код должен быть **клиентским** — **без сборки и запуска через bash**, без серверной части. Сайт должен быть deploy-ready на любом shared hosting как набор файлов (index.html, /css, /js, /exercises, /assets).
> 2. Разрешено подключение внешних библиотек **только через CDN** (скрипты/стили). Никаких npm/yarn/webpack/parcel.
> 3. Код должен быть модульным (ES6 модули), легко читаемый и **сильно прокомментирован** — т.к. сайт будут много править вручную.
> 4. Минимум зависимостей: использовать **native SVG** и vanilla JS. Можно подключить лёгкую библиотеку для SVG через CDN (например svg.js), но это опционально.
>
> **Функциональные требования (важно — реализовать):**
> 1. **Главная структура:** статическая навигация по главам (7/8/9 класс → главы учебника → темы → упражнения).
> 2. **Exercise engine:** каждое упражнение берётся из JSON (`/exercises/<id>.json`) и рендерится на странице. JSON описывает: мета (название, цель, ключевые параметры), svg-конфигурацию (точки, отрезки, прямые, окружности), интерактивные контролы (ползунки, чекбоксы, кнопки), сценарии проверки (условия правильности) и подсказки/шаги.
> 3. **Editor mode:** встроенный визуальный редактор JSON (WYSIWYG-панель + raw JSON), чтобы учитель мог создавать/менять упражнения без правки кода. Editor сохраняет JSON локально (download/upload) и показывает live preview.
> 4. **Интерактивные элементы:** перетаскиваемые точки, ползунки (input type=range), изменение углов/длин в реальном времени, подсветка равных углов/отрезков, показ измерений (градусы/единицы), snap-to-grid опция, режимы «демонстрация» и «задание» (ученик выполняет).
> 5. **UI/UX:** responsive, touch-friendly, поддержка клавиатуры. Использовать минимальный CSS-фреймворк (Bulma или Bootstrap через CDN) или чистый CSS.
> 6. **Примеры и тесты:** сгенерируй страницу `/examples.html` с 6–10 готовыми упражнениями (включая два приведённых мною примера). Каждый пример — его JSON + preview + ссылка «Скачать JSON», «Открыть в редакторе».
> 7. **Документация:** README.html (или README.md) с инструкцией, как добавлять упражнение, формат JSON, структура проекта, рекомендации по редактированию.
> 8. **Качество кода:** использовать JSDoc-комментарии, минимальные зависимости, корректная организация файлов (см. ниже).
>
> **Файловая структура (обязательно):**
> ```
> /index.html
> /README.md
> /css/
>   main.css
> /js/
>   app.js            // роутер + инициализация
>   renderer.js       // общий рендерер SVG
>   exercises-api.js  // загрузка/сохранение JSON, helper
>   editor.js         // визуальный редактор
>   widgets/          // draggable, slider, angle-measurer
>     draggable.js
>     slider.js
>     angle.js
> /exercises/
>   001-parallel-transversal.json
>   002-triangle-area.json
> /examples.html
> /assets/
>   icons/*.svg
> ```
>
> **JSON-схема упражнения (обязательное поле в README):** укажи пример JSON-схемы, примерно так:
> ```json
> {
>   "id": "001-parallel-transversal",
>   "title": "Параллельные прямые и секущая",
>   "class": "7",
>   "description": "Две параллельные прямые и секущая. Подсветить равные углы.",
>   "svg": {
>     "width": 800,
>     "height": 400,
>     "elements": [
>       {"type":"line","id":"l1","a":[50,60],"b":[750,60]},
>       {"type":"line","id":"l2","a":[50,300],"b":[750,300]},
>       {"type":"movable_line","id":"t","p1":[200,0],"p2":[400,400]}
>     ]
>   },
>   "controls": [
>     {"type":"slider","id":"angle","target":"t","prop":"rotation","min":-60,"max":60},
>     {"type":"checkbox","id":"show-degrees","label":"Показывать градусы"}
>   ],
>   "hints": ["Угол A равен углу B и т.д."],
>   "solution": {"type":"visual_equality","pairs":[["angle1","angle2"],["angle3","angle4"]]}

> ```
>
> **Обязательные UI-фичи упражнения:**
> - Автоматическая подсветка эквивалентных элементов (например, равные углы одним цветом).
> - Показ численных значений (градусы/длины) при включении «показывать» (чтобы учитель мог проверять).
> - Кнопка «Показать шаги/доказательство» — статический блок с текстом и иллюстрацией (из JSON).
>
> **ТЗ по двум примерам (реализовать обязательно):**
> - Пример 1 (параллельные и секущая) — интерактивная секущая: draggable line или slider для угла; все пары равных углов подсвечиваются одним цветом; подписи (∠) показываются по hover; значения в градусах меняются в реальном времени.
> - Пример 2 (треугольник в прямоугольнике) — прямоугольник фиксирован, внутри треугольник с общим основанием; высота от вершины треугольника до основания — регулируемый ползунок/drag вершины; показывать область треугольника и половину области прямоугольника, а также числовую проверку (area_triangle == 0.5*area_rectangle) при любом положении.
>
> **Критерии приёмки (что должно работать):**
> 1. Сайт разворачивается на простом шаред-хостинге без дополнительных шагов — просто разложить файлы.
> 2. JSON-файлы корректно загружаются/редактируются без правки кода.
> 3. Примеры работают на мобильных и десктопах, touch-drag корректно.
> 4. Проект сопровождается README с описанием JSON-схемы и инструкцией «как добавить новое упражнение».
>
> **Дополнительно (желательно):**

> - Простая система шаблонов для печати (PDF friendly).
> - Лёгкая локализация (ru/en) — строки в отдельном `i18n.json`.
>
> **Не делай:** серверный код, сборку, бандлеры, webpack, node scripts, back-end.

```json

    "width": 800,
```

## Набор интерактивных сценариев по главам (кратко)

### Глава I — Начальные геометрические сведения
- Точка/прямая/отрезок/луч — перетаскивай и наблюдай.
- Угол: измерение и перенос — draggable луч, показ градусов.
- Провешивание прямой (анимация).

### Глава III — Параллельные прямые (и секущие)
- Секущая и пары углов (реализовать обязательно).
- Угол между прямыми.
- Признаки параллельности.

### Глава II — Треугольники
- Классификация и периметр.
- Медианы, высоты, биссектрисы (автоматическое построение).
- Признаки равенства треугольников.
- Площадь треугольника (реализовать обязательно — демонстрация через прямоугольник).

### Глава IV — Соотношения между сторонами и углами
- Теорема Пифагора (наглядное доказательство через площади).
- Признаки подобия, пропорции.

### Многоугольники и параллелограммы
- Свойства диагоналей, средняя линия трапеции, сумма углов.

### Окружность
- Центр/радиус/хорда, вписанные/центральные углы, секущая-касательная.

### Преобразования и подобие
- Параллельный перенос, поворот, симметрия, гомотетия.

### Контрольные задания
- Автопроверка с допуском, многоуровневые подсказки, экспорт заданий.

## README (короткая инструкция для включения в проект)

1. Разверните файлы на шаред-хостинге — просто загрузите все файлы и откройте `index.html`.
2. Добавьте новое упражнение: создайте JSON по шаблону и положите в `/exercises/`.
3. Откройте `examples.html` или `editor.html` и загрузите новый JSON.
